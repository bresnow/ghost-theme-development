# {{THEME_NAME}}

A Ghost theme built with Vite and PostCSS.

## Requirements

- **Node.js 20+**
- **Ghost >= 6.0.0**

## Quick Start (standalone)

If you received this theme on its own, you can use it directly without any workspace setup:

```bash
npm install
npm run build
```

Then upload the theme to your Ghost site:
1. Go to your Ghost Admin panel (e.g. `https://your-site.com/ghost`)
2. Settings > Design > Change theme > Upload theme
3. Select the theme folder or zip it first: `zip -r {{THEME_NAME}}.zip assets/ partials/ locales/ *.hbs package.json`

## Development

```bash
npm install
npm run dev
```

This starts Vite in watch mode. Edit source files in `assets/css/` and `assets/js/` and Vite will rebuild `assets/built/` automatically.

## Scripts

| Command | What it does |
|---|---|
| `npm run dev` | Watch source files and rebuild on change |
| `npm run build` | One-time production build |
| `npm run lint` | Validate theme with gscan |

## Theme Structure

```
{{THEME_NAME}}/
├── assets/
│   ├── css/          # Source stylesheets
│   ├── js/           # Source scripts
│   └── built/        # Compiled output (generated by Vite)
├── partials/         # Reusable Handlebars partials
├── default.hbs       # Base layout
├── index.hbs         # Post list
├── post.hbs          # Single post
├── page.hbs          # Static page
├── error.hbs         # Error page
├── package.json      # Theme manifest + build config
├── vite.config.js    # Vite build configuration
└── postcss.config.js # PostCSS plugins (autoprefixer)
```

## Customizing

- **CSS**: Edit files in `assets/css/`. Vite processes them through PostCSS with autoprefixer.
- **JS**: Edit files in `assets/js/`. The entry point imports CSS so both are built from a single command.
- **Templates**: Edit `.hbs` files. If running inside Ghost with `NODE_ENV=development`, the browser auto-refreshes on template changes.

## Validating

Run gscan to check your theme against the Ghost theme spec before uploading:

```bash
npm run lint
```

Zero errors means Ghost will accept the theme. Warnings are non-blocking recommendations.
